# 📋 Case Study Requirements: New Project Details

This document outlines the specific data needed to add a new project to the portfolio. Each section corresponds to a part of the premium layout designed for **Projekts**.

---

## 1. The List View (Homepage)
*This info appears on the scrolling "Selected Work" list on the home page.*

- **Client Name**: The brand name (e.g., "Glitz & Glamour").
- **Category**: Industry/Service type (e.g., "Booking Platform").
- **Short Title**: A catchy one-liner (e.g., "End-to-end salon booking & loyalty platform").
- **Primary Metric**: A big number to grab attention (e.g., "+180%").
- **Metric Label**: What that number is (e.g., "Online Bookings").
- **Tags**: Tech stack (e.g., "Next.js", "AI", "Node.js").
- **Client Logo**: High-quality SVG/PNG (appears when the row is hovered).

---

## 2. The Detailed Case Study (Full Page)
*This info is for the dedicated project page.*

### A. The Hero Section
- **Main Headline**: A bold statement of transformation (e.g., "From Manual Workflows to a Fully Integrated Platform").
- **Subtitle**: 2-3 sentences explaining the "Big Picture" impact and the core value delivered.

### B. Success Metrics (Stats Bar)
Provide **4 key statistics** that prove success:
1. (e.g., "+200% User Engagement")
2. (e.g., "50+ Hours Saved/Month")
3. (e.g., "0 Manual Errors")
4. (e.g., "24/7 Availability")

### C. The Problem (The "Before")
- Narrative context: Why did they come to us?
- Specific pain points: What was breaking or slowing them down? (Manual work, lost data, scalability issues).

### D. The Solution (The "After")
- **Tech Stack**: Specific tools used (e.g., "Next.js, Node.js, Prisma, PostgreSQL").
- **Core Features (List 6 features)**: 
  - **Title**: (e.g., "AI Booking Agent")
  - **Description**: How it solved a specific part of the problem.

### E. The Flow (Step-by-Step Journey)
List **5-7 steps** of the final user experience:
1. **Discovery**: How the user enters the system.
2. **Action**: What they do.
3. **Outcome**: What happens automatically.
4. **Retention**: How the system keeps them coming back.

### F. Analytics (Dashboard Visibility)
- List the specific metrics the client can now see in their admin panel (e.g., "Peak Booking Hours", "Staff Utilization", "Revenue Trends").

### G. Final Result
- **4 Final Outcomes**: One-sentence summaries of the biggest wins.
- **Project Recap**: 
  - What was built.
  - What it replaced.
  - What it delivered.

---

## 3. Visual Assets Needed
- **Brand Logo**: High-res SVG or PNG.
- **Product Screenshots**: 5-8 screenshots showing different parts of the UI (Dashboard, Mobile view, Public landing pages).
- **Video/GIF (Optional)**: Short clips of specific interactions (e.g., the AI Agent chatting).

---

> [!IMPORTANT]
> **Focus on Transformation**: For every feature, explain not just *what* it is, but *how* it changed the business. Instead of "We added an SMS feature," use "Automated SMS reminders reduced no-shows by 40%."
